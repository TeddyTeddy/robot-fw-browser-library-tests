16"""A test library for handling date and time values. 

17 

18``DateTime`` is a Robot Framework standard library that supports creating and 

19converting date and time values (e.g. `Get Current Date`, `Convert Time`), 

20as well as doing simple calculations with them (e.g. `Subtract Time From Date`, 

21`Add Time To Time`). It supports dates and times in various formats, and can 

22also be used by other libraries programmatically. 

23 

24== Table of contents == 

25 

26%TOC% 

27 

28= Terminology = 

29 

30In the context of this library, ``date`` and ``time`` generally have following 

31meanings: 

32 

33- ``date``: An entity with both date and time components but without any 

34 timezone information. For example, ``2014-06-11 10:07:42``. 

35- ``time``: A time interval. For example, ``1 hour 20 minutes`` or ``01:20:00``. 

36 

37This terminology differs from what Python's standard 

38[http://docs.python.org/library/datetime.html|datetime] module uses. 

39Basically its 

40[http://docs.python.org/library/datetime.html#datetime-objects|datetime] and 

41[http://docs.python.org/library/datetime.html#timedelta-objects|timedelta] 

42objects match ``date`` and ``time`` as defined by this library. 

43 

44= Date formats = 

45 

46Dates can given to and received from keywords in `timestamp`, `custom 

47timestamp`, `Python datetime` and `epoch time` formats. These formats are 

48discussed thoroughly in subsequent sections. 

49 

50Input format is determined automatically based on the given date except when 

51using custom timestamps, in which case it needs to be given using 

52``date_format`` argument. Default result format is timestamp, but it can 

53be overridden using ``result_format`` argument. 

54 

55== Timestamp == 

56 

57If a date is given as a string, it is always considered to be a timestamp. 

58If no custom formatting is given using ``date_format`` argument, the timestamp 

59is expected to be in [http://en.wikipedia.org/wiki/ISO_8601|ISO 8601] like 

60format ``YYYY-MM-DD hh:mm:ss.mil``, where any non-digit character can be used 

61as a separator or separators can be omitted altogether. Additionally, 

62only the date part is mandatory, all possibly missing time components are 

63considered to be zeros. 

64 

65Dates can also be returned in the same ``YYYY-MM-DD hh:mm:ss.mil`` format by 

66using ``timestamp`` value with ``result_format`` argument. This is also the 

67default format that keywords returning dates use. Milliseconds can be excluded 

68using ``exclude_millis`` as explained in `Millisecond handling` section. 

69 

70Examples: 

71| ${date1} = | Convert Date | 2014-06-11 10:07:42.000 | 

72| ${date2} = | Convert Date | 20140611 100742 | result_format=timestamp | 

73| Should Be Equal | ${date1} | ${date2} | 

74| ${date} = | Convert Date | 20140612 12:57 | exclude_millis=yes | 

75| Should Be Equal | ${date} | 2014-06-12 12:57:00 | 

76 

77== Custom timestamp == 

78 

79It is possible to use custom timestamps in both input and output. 

80The custom format is same as accepted by Python's 

81[http://docs.python.org/library/datetime.html#strftime-strptime-behavior| 

82datetime.strptime] function. For example, the default timestamp discussed 

83in the previous section would match ``%Y-%m-%d %H:%M:%S.%f``. 

84 

85When using a custom timestamp in input, it must be specified using 

86``date_format`` argument. The actual input value must be a string that matches 

87the specified format exactly. When using a custom timestamp in output, it must 

88be given using ``result_format`` argument. 

89 

90Examples: 

91| ${date} = | Convert Date | 28.05.2014 12:05 | date_format=%d.%m.%Y %H:%M | 

92| Should Be Equal | ${date} | 2014-05-28 12:05:00.000 | 

93| ${date} = | Convert Date | ${date} | result_format=%d.%m.%Y | 

94| Should Be Equal | ${date} | 28.05.2014 | 

95 

96Notice that locale aware directives like ``%b`` do not work correctly with 

97Jython on non-English locales: http://bugs.jython.org/issue2285 

98 

99== Python datetime == 

100 

101Python's standard 

102[http://docs.python.org/library/datetime.html#datetime-objects|datetime] 

103objects can be used both in input and output. In input they are recognized 

104automatically, and in output it is possible to get them by giving ``datetime`` 

105value to ``result_format`` argument. 

106 

107One nice benefit with datetime objects is that they have different time 

108components available as attributes that can be easily accessed using the 

109extended variable syntax. 

110 

111Examples: 

112| ${datetime} = | Convert Date | 2014-06-11 10:07:42.123 | datetime | 

113| Should Be Equal As Integers | ${datetime.year} | 2014 | 

114| Should Be Equal As Integers | ${datetime.month} | 6 | 

115| Should Be Equal As Integers | ${datetime.day} | 11 | 

116| Should Be Equal As Integers | ${datetime.hour} | 10 | 

117| Should Be Equal As Integers | ${datetime.minute} | 7 | 

118| Should Be Equal As Integers | ${datetime.second} | 42 | 

119| Should Be Equal As Integers | ${datetime.microsecond} | 123000 | 

120 

121== Epoch time == 

122 

123Epoch time is the time in seconds since the 

124[http://en.wikipedia.org/wiki/Unix_time|UNIX epoch] i.e. 00:00:00.000 (UTC) 

1251 January 1970. To give a date in epoch time, it must be given as a number 

126(integer or float), not as a string. To return a date in epoch time, 

127it is possible to use ``epoch`` value with ``result_format`` argument. 

128Epoch time is returned as a floating point number. 

129 

130Notice that epoch time itself is independent on timezones and thus same 

131around the world at a certain time. What local time a certain epoch time 

132matches obviously then depends on the timezone. For example, examples below 

133were tested in Finland but verifications would fail on other timezones. 

134 

135Examples: 

136| ${date} = | Convert Date | ${1000000000} | 

137| Should Be Equal | ${date} | 2001-09-09 04:46:40.000 | 

138| ${date} = | Convert Date | 2014-06-12 13:27:59.279 | epoch | 

139| Should Be Equal | ${date} | ${1402568879.279} | 

140 

141== Earliest supported date == 

142 

143The earliest date that is supported depends on the date format and to some 

144extent on the platform: 

145 

146- Timestamps support year 1900 and above. 

147- Python datetime objects support year 1 and above. 

148- Epoch time supports 1970 and above on Windows with Python and IronPython. 

149- On other platforms epoch time supports 1900 and above or even earlier. 

150 

151= Time formats = 

152 

153Similarly as dates, times can be given to and received from keywords in 

154various different formats. Supported formats are `number`, `time string` 

155(verbose and compact), `timer string` and `Python timedelta`. 

156 

157Input format for time is always determined automatically based on the input. 

158Result format is number by default, but it can be customised using 

159``result_format`` argument. 

160 

161== Number == 

162 

163Time given as a number is interpreted to be seconds. It can be given 

164either as an integer or a float, or it can be a string that can be converted 

165to a number. 

166 

167To return a time as a number, ``result_format`` argument must have value 

168``number``, which is also the default. Returned number is always a float. 

169 

170Examples: 

171| ${time} = | Convert Time | 3.14 | 

172| Should Be Equal | ${time} | ${3.14} | 

173| ${time} = | Convert Time | ${time} | result_format=number | 

174| Should Be Equal | ${time} | ${3.14} | 

175 

176== Time string == 

177 

178Time strings are strings in format like ``1 minute 42 seconds`` or ``1min 42s``. 

179The basic idea of this format is having first a number and then a text 

180specifying what time that number represents. Numbers can be either 

181integers or floating point numbers, the whole format is case and space 

182insensitive, and it is possible to add a minus prefix to specify negative 

183times. The available time specifiers are: 

184 

185- ``days``, ``day``, ``d`` 

186- ``hours``, ``hour``, ``h`` 

187- ``minutes``, ``minute``, ``mins``, ``min``, ``m`` 

188- ``seconds``, ``second``, ``secs``, ``sec``, ``s`` 

189- ``milliseconds``, ``millisecond``, ``millis``, ``ms`` 

190 

191When returning a time string, it is possible to select between ``verbose`` 

192and ``compact`` representations using ``result_format`` argument. The verbose 

193format uses long specifiers ``day``, ``hour``, ``minute``, ``second`` and 

194``millisecond``, and adds ``s`` at the end when needed. The compact format uses 

195shorter specifiers ``d``, ``h``, ``min``, ``s`` and ``ms``, and even drops 

196the space between the number and the specifier. 

197 

198Examples: 

199| ${time} = | Convert Time | 1 minute 42 seconds | 

200| Should Be Equal | ${time} | ${102} | 

201| ${time} = | Convert Time | 4200 | verbose | 

202| Should Be Equal | ${time} | 1 hour 10 minutes | 

203| ${time} = | Convert Time | - 1.5 hours | compact | 

204| Should Be Equal | ${time} | - 1h 30min | 

205 

206== Timer string == 

207 

208Timer string is a string given in timer like format ``hh:mm:ss.mil``. In this 

209format both hour and millisecond parts are optional, leading and trailing 

210zeros can be left out when they are not meaningful, and negative times can 

211be represented by adding a minus prefix. 

212 

213To return a time as timer string, ``result_format`` argument must be given 

214value ``timer``. Timer strings are by default returned in full ``hh:mm:ss.mil`` 

215format, but milliseconds can be excluded using ``exclude_millis`` as explained 

216in `Millisecond handling` section. 

217 

218Examples: 

219| ${time} = | Convert Time | 01:42 | 

220| Should Be Equal | ${time} | ${102} | 

221| ${time} = | Convert Time | 01:10:00.123 | 

222| Should Be Equal | ${time} | ${4200.123} | 

223| ${time} = | Convert Time | 102 | timer | 

224| Should Be Equal | ${time} | 00:01:42.000 | 

225| ${time} = | Convert Time | -101.567 | timer | exclude_millis=yes | 

226| Should Be Equal | ${time} | -00:01:42 | 

227 

228== Python timedelta == 

229 

230Python's standard 

231[http://docs.python.org/library/datetime.html#datetime.timedelta|timedelta] 

232objects are also supported both in input and in output. In input they are 

233recognized automatically, and in output it is possible to receive them by 

234giving ``timedelta`` value to ``result_format`` argument. 

235 

236Examples: 

237| ${timedelta} = | Convert Time | 01:10:02.123 | timedelta | 

238| Should Be Equal | ${timedelta.total_seconds()} | ${4202.123} | 

239 

240= Millisecond handling = 

241 

242This library handles dates and times internally using the precision of the 

243given input. With `timestamp`, `time string`, and `timer string` result 

244formats seconds are, however, rounded to millisecond accuracy. Milliseconds 

245may also be included even if there would be none. 

246 

247All keywords returning dates or times have an option to leave milliseconds out 

248by giving a true value to ``exclude_millis`` argument. If the argument is given 

249as a string, it is considered true unless it is empty or case-insensitively 

250equal to ``false``, ``none`` or ``no``. Other argument types are tested using 

251same [http://docs.python.org/library/stdtypes.html#truth|rules as in 

252Python]. 

253 

254When milliseconds are excluded, seconds in returned dates and times are 

255rounded to the nearest full second. With `timestamp` and `timer string` 

256result formats, milliseconds will also be removed from the returned string 

257altogether. 

258 

259Examples: 

260| ${date} = | Convert Date | 2014-06-11 10:07:42 | 

261| Should Be Equal | ${date} | 2014-06-11 10:07:42.000 | 

262| ${date} = | Convert Date | 2014-06-11 10:07:42.500 | exclude_millis=yes | 

263| Should Be Equal | ${date} | 2014-06-11 10:07:43 | 

264| ${dt} = | Convert Date | 2014-06-11 10:07:42.500 | datetime | exclude_millis=yes | 

265| Should Be Equal | ${dt.second} | ${43} | 

266| Should Be Equal | ${dt.microsecond} | ${0} | 

267| ${time} = | Convert Time | 102 | timer | exclude_millis=false | 

268| Should Be Equal | ${time} | 00:01:42.000 | | 

269| ${time} = | Convert Time | 102.567 | timer | exclude_millis=true | 

270| Should Be Equal | ${time} | 00:01:43 | | 

271 

272= Programmatic usage = 

273 

274In addition to be used as normal library, this library is intended to 

275provide a stable API for other libraries to use if they want to support 

276same date and time formats as this library. All the provided keywords 

277are available as functions that can be easily imported: 

278 

279| from robot.libraries.DateTime import convert_time 

280| 

281| def example_keyword(timeout): 

282| seconds = convert_time(timeout) 

283| # ... 

284 

285Additionally helper classes ``Date`` and ``Time`` can be used directly: 

286 

287| from robot.libraries.DateTime import Date, Time 

288| 

289| def example_keyword(date, interval): 

290| date = Date(date).convert('datetime') 

291| interval = Time(interval).convert('number') 

292| # ... 

293"""

312def get_current_date(time_zone='local', increment=0, 

313 result_format='timestamp', exclude_millis=False): 

314 """Returns current local or UTC time with an optional increment. 

315 

316 Arguments: 

317 - ``time_zone:`` Get the current time on this time zone. Currently only 

318 ``local`` (default) and ``UTC`` are supported. 

319 - ``increment:`` Optional time increment to add to the returned date in 

320 one of the supported `time formats`. Can be negative. 

321 - ``result_format:`` Format of the returned date (see `date formats`). 

322 - ``exclude_millis:`` When set to any true value, rounds and drops 

323 milliseconds as explained in `millisecond handling`. 

324 

325 Examples: 

326 | ${date} = | Get Current Date | 

327 | Should Be Equal | ${date} | 2014-06-12 20:00:58.946 | 

328 | ${date} = | Get Current Date | UTC | 

329 | Should Be Equal | ${date} | 2014-06-12 17:00:58.946 | 

330 | ${date} = | Get Current Date | increment=02:30:00 | 

331 | Should Be Equal | ${date} | 2014-06-12 22:30:58.946 | 

332 | ${date} = | Get Current Date | UTC | - 5 hours | 

333 | Should Be Equal | ${date} | 2014-06-12 12:00:58.946 | 

334 | ${date} = | Get Current Date | result_format=datetime | 

335 | Should Be Equal | ${date.year} | ${2014} | 

336 | Should Be Equal | ${date.month} | ${6} | 

337 """ 

338 upper = time_zone.upper() 

339 if upper == 'LOCAL': 

340 dt = datetime.now() 

341 # Epoch time is same regardless the timezone. We convert `dt` to epoch time 

342 # using `time.mktime()` afterwards, and it expects time in local time. 

343 # For details: https://github.com/robotframework/robotframework/issues/3306 

344 elif upper == 'UTC' and result_format.upper() == 'EPOCH': 

345 dt = datetime.now() 

346 elif upper == 'UTC': 

347 dt = datetime.utcnow() 

348 else: 

349 raise ValueError("Unsupported timezone '%s'." % time_zone) 

350 date = Date(dt) + Time(increment) 

351 return date.convert(result_format, millis=is_falsy(exclude_millis))

516 def __init__(self, date, input_format=None): 

517 self.datetime = self._convert_to_datetime(date, input_format) 

518 

519 @property 

520 def seconds(self): 

521 # Mainly for backwards compatibility with RF 2.9.1 and earlier. 

522 return self._convert_to_epoch(self.datetime) 

523 

524 def _convert_to_datetime(self, date, input_format): 

525 if isinstance(date, datetime): 

526 return date 

527 if is_number(date): 

528 return self._seconds_to_datetime(date) 

529 if is_string(date): 

530 return self._string_to_datetime(date, input_format) 

531 raise ValueError("Unsupported input '%s'." % date)

533 def _seconds_to_datetime(self, secs): 

534 # Workaround microsecond rounding errors with IronPython: 

535 # https://github.com/IronLanguages/main/issues/1170 

536 # Also Jython had similar problems, but they seem to be fixed in 2.7. 

537 dt = datetime.fromtimestamp(secs) 

538 return dt.replace(microsecond=roundup(secs % 1 * 1e6))

540 def _string_to_datetime(self, ts, input_format): 

541 if not input_format: 

542 ts = self._normalize_timestamp(ts) 

543 input_format = '%Y-%m-%d %H:%M:%S.%f' 

544 if self._need_to_handle_f_directive(input_format): 

545 return self._handle_un_supported_f_directive(ts, input_format) 

546 return datetime.strptime(ts, input_format) 

547 

548 def _normalize_timestamp(self, date): 

549 ts = ''.join(d for d in date if d.isdigit()) 

550 if not (8 <= len(ts) <= 20): 

551 raise ValueError("Invalid timestamp '%s'." % date) 

552 ts = ts.ljust(20, '0') 

553 return '%s-%s-%s %s:%s:%s.%s' % (ts[:4], ts[4:6], ts[6:8], ts[8:10], 

554 ts[10:12], ts[12:14], ts[14:]) 

555 

556 def _need_to_handle_f_directive(self, format): 

557 # https://github.com/IronLanguages/main/issues/1169 

558 return IRONPYTHON and '%f' in format 

559 

560 def _handle_un_supported_f_directive(self, ts, input_format): 

561 input_format = self._remove_f_from_format(input_format) 

562 match = re.search(r'\d+$', ts) 

563 if not match: 

564 raise ValueError("time data '%s' does not match format '%s%%f'." 

565 % (ts, input_format)) 

566 end_digits = match.group(0) 

567 micro = int(end_digits.ljust(6, '0')) 

568 dt = datetime.strptime(ts[:-len(end_digits)], input_format) 

569 return dt.replace(microsecond=micro) 

570 

571 def _remove_f_from_format(self, format): 

572 if not format.endswith('%f'): 

573 raise ValueError('%f directive is supported only at the end of ' 

574 'the format string on this Python interpreter.') 

575 return format[:-2]

577 def convert(self, format, millis=True): 

578 dt = self.datetime 

579 if not millis: 

580 secs = 1 if dt.microsecond >= 5e5 else 0 

581 dt = dt.replace(microsecond=0) + timedelta(seconds=secs) 

582 if '%' in format: 

583 return self._convert_to_custom_timestamp(dt, format) 

584 format = format.lower() 

585 if format == 'timestamp': 

586 return self._convert_to_timestamp(dt, millis) 

587 if format == 'datetime': 

588 return dt 

589 if format == 'epoch': 

590 return self._convert_to_epoch(dt) 

591 raise ValueError("Unknown format '%s'." % format) 

592 

593 def _convert_to_custom_timestamp(self, dt, format): 

594 if not self._need_to_handle_f_directive(format): 

595 return dt.strftime(format) 

596 format = self._remove_f_from_format(format) 

597 return dt.strftime(format) + '%06d' % dt.microsecond 

598 

599 def _convert_to_timestamp(self, dt, millis=True): 

600 if not millis: 

601 return dt.strftime('%Y-%m-%d %H:%M:%S') 

602 ms = roundup(dt.microsecond / 1000.0) 

603 if ms == 1000: 

604 dt += timedelta(seconds=1) 

605 ms = 0 

606 return dt.strftime('%Y-%m-%d %H:%M:%S') + '.%03d' % ms

608 def _convert_to_epoch(self, dt): 

609 return time.mktime(dt.timetuple()) + dt.microsecond / 1e6
